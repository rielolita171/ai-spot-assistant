UX & Target Audience Context

Target: Remote workers, cafe-hoppers, and privacy-conscious users running self-hosted environments.

UX Strategy: Mobile-first approach utilizing Capacitor for Android native feel. Components use bottom-sheets for reachability, skeleton loaders to mask AI latency, and local storage (SQLite) to ensure the app functions robustly even without internet access.

Key User Story: Intelligent Discovery

"As a user, I want to search for a 'cozy and bright afternoon' spot so I can take good photos while working."

→ Triggers Vector Search on ChromaDB.

Database Topography

The system implements a dual-database paradigm. The structured, transactional data (receipts, WiFi speeds) is handled by a standard relational schema. Semantic, conversational data used by the AI for reasoning is stored in a Vector DB. Select a paradigm below to explore the tables/collections.

Note: ChromaDB does not use relational tables. It uses Collections of embedded text. The backend synthesizes structured tags into descriptive paragraphs before embedding, allowing the AI to search by meaning (e.g., finding "halal, quiet" locations via mathematical vector similarity).
